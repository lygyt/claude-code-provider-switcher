# Claude Code Provider Switcher — 安装使用说明

这是一个 VS Code 扩展，能让你在 **Anthropic、智谱 GLM、DeepSeek、Kimi、通义千问、MiniMax** 等多个 AI 服务商之间一键切换，并在一个侧边栏里管理 **技能（Skills）** 和 **多模型协作工作流（Workflows）**。

最大的亮点是 **工作流**：你可以把多个 AI 模型串成一条流水线，让它们像团队一样协作完成一个任务——比如「研究员写简报 → 架构师出方案 → 编码员写代码 → 测试员审代码 → 写作员写论文 → 同行评审打回修改」，还能设置 **循环规则**，让评审不通过时自动退回重做，直到满意或达到次数上限。

---

## 一、你需要准备什么

| 必需 | 说明 |
|---|---|
| **VS Code** | 版本 1.93 或以上 |
| **一个 API Key** | 比如 DeepSeek、智谱 GLM 的 key（用来调用模型） |

| 可选 | 说明 |
|---|---|
| **Claude Code CLI** | 如果你想用扩展「启动一个 Claude Code 终端」功能，需要先装好 `claude` 命令并登录。**只用工作流功能则不需要。** |
| **Node.js** | 只在「想从源码重新打包」时才需要。**直接装 VSIX 不需要 Node。** |
| **Python 环境** | 工作流里的「编码员」角色会生成 Python 代码，但扩展本身不会执行它（由你或 AI 决定怎么跑）。 |

## 二、安装（两种方式，任选其一）

### 方式 A：直接装预编译包（推荐，最简单）

文件夹里已经有一个打包好的文件：

```
claude-code-provider-switcher-0.1.0.vsix
```

1. 打开 VS Code。
2. 在左侧「扩展」面板里，点右上角的 **`⋯`** → **「从 VSIX 安装…」(Install from VSIX…)**。
3. 选中上面的 `.vsix` 文件，安装。
4. 安装完成后，**重新加载窗口**：按 `Ctrl+Shift+P`（Mac 是 `Cmd+Shift+P`），输入 `Reload Window` 回车。

装完后，VS Code 左侧活动栏会出现一个 **Claude Code** 图标 ✨，点开就是它的侧边栏。

### 方式 B：从源码自己打包（想改代码或要最新版时用）

需要先装好 Node.js（16 以上）。在本文件夹里打开终端：

```bash
npm install        # 装依赖（第一次才需要）
npm run package    # 编译 + 测试 + 打包，生成 claude-code-provider-switcher-0.1.0.vsix
```

然后用方式 A 安装生成的 VSIX 即可。

> ⚠️ 如果 `npm` 命令报错说找不到 `node`，说明你的终端 PATH 里没有 Node，需要先把 Node 加到环境变量，或用完整路径调用。

## 三、首次配置：添加服务商和密钥

1. 点左侧活动栏的 **Claude Code** 图标，展开 **Claude Code Providers** 面板。
2. 下拉框里已经内置了几个预设（DeepSeek、智谱 GLM、Kimi 等）。选一个，比如 **DeepSeek**。
3. 在 **API Key / Token** 框里粘贴你的 DeepSeek key（形如 `sk-xxxx...`），点 **Save Key**。旁边出现「Saved」就存好了。
   - 密钥保存在 `~/.claude-code-provider-switcher/config.json`，不会进 Git，也不会上传任何地方。
4. 想加自己的服务商？按 `Ctrl+Shift+P` → 输入 `Claude Code: Add Provider`，按提示填名称、鉴权方式、Base URL 即可。

## 四、核心功能怎么用

侧边栏从上到下有四个面板：

```
Claude Code Providers   ← 选服务商、存密钥、启动终端
Claude Code History     ← 看历史会话、一键 resume
Claude Code Skills      ← 管理技能
Claude Code Workflows   ← 多模型协作工作流（重点）
```

### 4.1 切换服务商 & 启动 Claude Code

- 在 Providers 下拉里选一个服务商，如果设置里开了「选择后自动启动」（默认开），会立刻开一个新终端跑 `claude`，并自动注入这个服务商的环境变量。
- 也可以手动启动：`Ctrl+Shift+P` → `Claude Code: Launch With Current Provider`。
- 多个服务商的终端可以同时开着，互不影响。

### 4.2 权限模式（侧边栏下拉）

Providers 面板里有个 **Permission Mode** 下拉：

| 选项 | 含义 |
|---|---|
| `Request Approval (default)` | 正常模式，敏感操作要你点确认 |
| `Full Access` | 免确认，终端以 `--permission-mode bypassPermissions` 启动 |

选了会自动记住，下次启动生效。

### 4.3 技能（Skills）管理

技能就是一段可复用的 `SKILL.md` 指令包，能让 Claude 在特定任务上更专业。

- **点一下技能** = 把它设为「活跃技能」（打星标 ⭐），新开的 Claude 会话会自动套用。
- **再点一下** = 取消活跃（取消星标）。
- 标题栏的 ⊘ 按钮 = 一键清除活跃技能（不清除已安装的后台技能）。
- **导入 GitHub 技能**：标题栏的 ☁ 按钮，粘一个 GitHub 仓库地址即可。
- 技能默认放在 `~/.claude/skills/<技能名>/`。

### 4.4 工作流（Workflows）—— 重头戏

工作流 = 一串「步骤」，每一步用**一个模型 + 可选一个技能**，上一步的产出自动喂给下一步，像流水线一样协作。

#### 模板变量（写在步骤的 Prompt 里）

| 变量 | 含义 |
|---|---|
| `{{task}}` | 你运行工作流时输入的原始任务 |
| `{{previous}}` | 上一步的输出（第一步为空） |
| `{{iteration}}` | 当前循环是第几轮（从 1 开始） |
| `{{review}}` | 最近一次评审的反馈（循环里才有） |

#### 新建一个工作流

1. 在 **Claude Code Workflows** 面板标题栏点 **`+`**，打开可视化编辑器。
2. 填工作流名称。
3. 点 **`+ Add Step`** 加步骤。每个步骤填：
   - **Step name**：角色名，比如「研究员」
   - **Provider**：用哪个服务商
   - **Model**：用哪个模型（选 Provider 后会自动从 API 拉取可选模型列表）
   - **Skill**（可选）：挂一个技能，它的 `SKILL.md` 会作为系统提示注入
   - **Prompt**：这一步要干什么，用 `{{task}}` / `{{previous}}` 串联
4. 用 ↑ ↓ 调整步骤顺序，✕ 删除。
5. （可选）勾选 **Enable primary review loop** 配一个循环：选「从哪一步重做」和「哪一步是评审」，设最大轮数。评审说 `Decision: APPROVED` 就通过，否则退回重做。
6. 点 **Save Workflow**。

#### 运行工作流

- 在某个工作流上点 ▶（或右键 → Run），输入任务描述。
- 扩展会按顺序逐步执行，右下角有进度条。
- 跑完自动打开一份 **Markdown 报告**，里面每一步的模型、技能、提示、输出都列得清清楚楚，还有循环的轮数和是否通过。

#### 循环（Loop）是怎么工作的

- 一个工作流可以配**多个不重叠的循环**。比如：编码↔测试 一个循环，写作↔同行评审 另一个循环。
- 评审步骤要在 prompt 里约定第一行写 `Decision: APPROVED` 或 `Decision: REVISE`。
- 不通过就自动退回「重做起点」重跑，最多到你设的轮数。
- **即使循环用尽次数还没通过，流水线也会继续往下走**（用当前最好的成果），不会卡死——这就是「自主完成」。
- 多个循环各自独立计数，互不干扰。

## 五、一键复现「科研 Vibe Coding」工作流

文件夹里自带一个脚本，能直接生成那条 8 角色、2 循环的科研全流程工作流（文献综述 → 研究设计 → 架构 → 编码↔测试循环 → 统计审稿 → 写作↔同行评审循环）。

### 准备技能（第一次才需要）

这个工作流用到了 6 个技能，需要先导入。在 **Claude Code Skills** 面板点 ☁ 按钮，分别粘这两个 GitHub 地址导入：

```
https://github.com/Imbad0202/academic-research-skills
https://github.com/K-Dense-AI/scientific-agent-skills
```

导入时选「Install after import」即可。这会装好：Deep Research、Academic Paper Reviewer、Literature Review、Exploratory Data Analysis、Statistical Analysis、Scientific Writing 等技能。

> 不想跑脚本？也可以在 Workflows 面板点 `+` 手动照着第五节的角色和 prompt 建一个。

### 生成工作流

需要 Node.js。在本文件夹打开终端：

```bash
node scripts/research-workflow-save.cjs
```

脚本会自动从你的 `skills.json` 里按名字找到这些技能，拼好 8 个步骤和 2 个循环，存进工作流。看到 `Saved workflow: ...` 就成功了。

然后回到 VS Code，**Claude Code Workflows** 面板里会出现「科研 Vibe Coding 协作流水线 v2」，点 ▶ 运行，输入一个科研任务（比如「研究咖啡因摄入与代码错误率的关系」），等它跑完看报告。

> 这一步只生成工作流定义；运行工作流用的是扩展内置的模型调用，不需要 Claude Code CLI。

## 六、配置文件都在哪

| 文件 | 作用 |
|---|---|
| `~/.claude-code-provider-switcher/config.json` | 服务商列表、密钥、当前选中的服务商 |
| `~/.claude-code-provider-switcher/skills.json` | 技能配置、已安装技能、分组 |
| `~/.claude-code-provider-switcher/workflows.json` | 所有工作流定义（含循环） |
| `~/.claude/skills/<技能名>/` | 技能的文件夹（含 SKILL.md） |

> `~` 是你的用户目录。Windows 上是 `C:\Users\你的用户名`。如果开了工作区，部分文件会优先放在工作区文件夹里。

想直接改配置也行——`Ctrl+Shift+P` → `Claude Code: Open Provider Config` / `Open Skill Config` 就能用编辑器打开。

## 七、常见问题

**Q：装完侧边栏没有 Claude Code 图标？**
A：按 `Ctrl+Shift+P` → `Reload Window` 重新加载一次。

**Q：选服务商后没自动开终端 / 报「Claude Code CLI was not found」？**
A：你没装 `claude` 命令，或它不在 PATH 里。要么装好 Claude Code CLI，要么关掉设置里的 `checkClaudeCliBeforeLaunch`，或者干脆只用工作流功能（工作流直接调 API，不需要 CLI）。

**Q：切权限模式时报 `Unexpected end of JSON input`？**
A：你的 VS Code `settings.json` 文件是空的或坏了。打开 `C:\Users\你的用户名\AppData\Roaming\Code\User\settings.json`，把内容改成合法的 JSON（空文件就改成 `{}` 然后保存），再试。新版扩展已对这种情况做了容错。

**Q：工作流跑到一半某步报错怎么办？**
A：扩展会停在出错那步，报告里会标出错误。最常见原因是：该服务商没存 API Key，或模型名写错了。检查 Provider 和 Model 再跑。

**Q：工作流的测试员角色真的会跑代码吗？**
A：不会真跑。当前测试员是「写测试用例 + 在脑子里逐步推演执行」的静态审查。如果想要真实执行 Python 测试并拿失败结果驱动回流，那是下一步可以加的功能。

**Q：循环一直不通过、跑满轮数怎么办？**
A：达到上限后流水线会带着当前最好的成果继续往下走（不会卡死）。想让它多试几轮，编辑工作流把「Max reviews」调大（最多 20）。

**Q：模型回答里带一堆「让我想想…」的思考过程？**
A：扩展在工作流里默认只取模型的最终回答，不把思考链传给下一步。单步输出里你看到的思考是 DeepSeek 等模型自带的，不影响下一步。

**Q：密钥安全吗？**
A：密钥只存在你本机的 `config.json`，扩展不上传、不收集遥测、不改你的 shell 配置、不覆盖 `~/.claude/settings.json`。别把 `config.json` 截图或提交到 Git 就行。
